# 需求分析

产品需求是走在具体开发前面，只有评审合格后的产品需求才能交付进一步开发。

最重要的是：`产品需求一定是可用的，明确定`。

不要需求都没有确定，就开始着手开发，那样会进入无止境的修改推脱责任中...

把不靠谱（不可用)的需求留在后面做，可能到时候需求就会更改。


## 产品设计5大 

首先整个产品设计分为5个大的方面，由低到高，底层根本原因决定上层形态构成。和网站分层也很形象。

    1. 战略层：我们为什么要做这个产品？

    2. 范围层：我们要建设什么样的内容？

    3. 结构层：如何呈现给用户？模式、顺序、优先级？

    4. 框架层：哪些功能在哪些页面上完成？这些功能在页面上如何实现？

    5. 表现层：不仅是美化，而是产品功能是否得到很好的展示？
