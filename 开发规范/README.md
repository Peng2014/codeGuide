#### 重要思想规范
<a id="main"></a>

1. 保持代码易于维护
2. 保持代码清晰易懂
3. 保持代码可拓展性




#### 文件和目录命名规范
<a id="host"></a>

1. 只能出现`小写英文字母`，`数字`和`横杠( - )`，以英语单词优先，多个单词以横杠 ( - ) 连接。
2. 尽量避免采用以下词汇来命名： ad、ads、adv、banner、sponsor、gg、guangg、guanggao。
3. 该规范适用于所有前端维护的内容和相关目录。(smarty, html, less, css, js, png, gif, jpg)。

例如：
* 项目命名：my-project-name ;
* 目录命名，静态资源名称都统一，例如：img,ui,widget;
* HTML文件命名 : error-report.html
* CSS,LESS文件命名:retina-sprites.less;
* js文件命名 : account-model.js;
